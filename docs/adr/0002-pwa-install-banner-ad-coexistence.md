# ADR-0002: PWAインストールバナーと広告の共存方法

## ステータス

承認済み

## コンテキスト

モバイルかつ未インストールのユーザーに、PWAのインストールを促す下部固定バナーを表示する。
対象ページのひとつである車中泊スポット詳細ページ（`/shachu-haku/[spotId]`）には、コンテンツ最下部に記事内のAdSense広告ユニット（`ADSENSE_SLOT_DETAIL_BOTTOM`）が存在する。

下部固定バナーは、最下部までスクロールしたときにこの広告と物理的に重なる。これは次の問題を引き起こす。

- AdSenseのポリシー（フローティング要素で広告を覆ってはいけない）への抵触
- 広告の視認性低下による収益への悪影響

干渉を避ける方法として、次の案があった。

- **案A**: バナー表示中だけ、コンテンツ下端にバナー高さ分の `padding-bottom` を動的に追加し、広告を含む全コンテンツをバナーより上にずらす
- **案B**: バナー表示中は最下部広告（または全広告）を非表示にする

## 決定

**案A（表示中のみ `padding-bottom` を追加し、広告は消さない）** を採用する。

## 理由

- 広告を出したまま一切覆わずに済み、AdSenseポリシーに適合する
- 広告価値の高い車中泊スポット詳細ページで、バナー表示中の収益機会を失わない
- 広告ユニットをアンマウント／再マウントすると `adsbygoogle.push({})` のライフサイクルで「already have ads」エラーや無効トラフィック扱いのリスクが生じる。案Aはこれを回避する
- 広告の表示・非表示の切り替えによるレイアウトシフト（CLS悪化）を避けられる。PageSpeed / Core Web Vitalsを重視する方針に沿う

## トレードオフ

バナー表示中はページ下端にバナー高さ分の余白が生じる。ただしバナーは却下または離脱で消え、余白もそれに連動して消えるため、影響は一時的かつ軽微である。

## 注意

この `padding-bottom` ロジックは「バナーが広告を覆わないための安全装置」である。安易に削除すると、最下部広告がバナーに覆われ、AdSenseポリシー違反とCLS悪化を再発させる。変更時はこのADRを参照すること。
