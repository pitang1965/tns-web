# ADR-0007: 車載対応はスマホ側ハンドオフ方式とし、CarPlay/Android Autoネイティブ対応は行わない

## ステータス

承認済み

## コンテキスト

モバイル展開の第2段階として「旅程から次の目的地へのナビをすぐ開始できる」車載向けアプリ(第1段階の獲得アプリとは別アプリ)を計画している。当初構想は「旅程からGoogle Maps検索をかけるだけのAndroid Auto/Apple CarPlay対応アプリ」だった。

調査の結果、この構想には非自明な技術的制約があることが判明した。

- **CarPlayアプリから車載画面上でGoogle Mapsを起動することはできない**。CarPlayアプリは他アプリを呼び出せず、POIテンプレートの経路ボタンが渡す先はApple Mapsのみ
- CarPlay対応にはAppleへのエンタイトルメント申請(カテゴリ審査)が必要で、ニッチな個人開発アプリには却下リスクと待ち時間がある
- Android AutoのPOIカテゴリアプリはGoogle Mapsへのナビインテント発行が可能だが、Android限定であり日本のiPhoneユーザー(約4〜5割)に届かない
- 一方、**スマホがCarPlay/Android Autoに接続された状態でスマホ側からGoogle Mapsのナビを開始すれば、車載画面に自動的に投影される**。Webの既存機能(全体ルート検索/途中からのルート検索)は既にこの経路で車載画面にナビを出せている

## 決定

車載画面向けのネイティブUI(CarPlay/Android Autoテンプレートアプリ)は作らない。代わりに**スマホ側ハンドオフアプリ**を作る:当日の旅程のアクティビティを大きなボタンで一覧表示し、タップでGoogle Mapsのナビを起動する。車載画面への表示はOS標準の投影に任せる。

## 理由

- 「旅程→Google Mapsナビ→車載画面」というユーザーが得たい結果を、エンタイトルメント申請なし・両OS対応・単一実装で実現できる
- CarPlayの制約(Google Maps起動不可)のもとでは、ネイティブCarPlay対応はApple Mapsへの妥協を強いられ、当初目的と矛盾する
- ソロ開発でWeb・獲得アプリと並行維持する前提で、投資対効果が最も高い

## トレードオフ

- 車載画面上で直接スポットや旅程を「閲覧」することはできない(操作は必ずスマホ側)。運転中の操作は想定せず、停車中の利用を前提とする
- 将来Android Autoの車載UIが本当に必要になった場合は、本ADRを見直したうえでAndroid限定のPOIアプリを追加検討する

## 注意

「CarPlay対応すればもっと便利では?」という発想が再浮上したら、まず本ADRのコンテキスト(Google Maps起動不可)が現在も有効かを確認すること。Appleがこの制約を緩和した場合のみ再検討の価値がある。
